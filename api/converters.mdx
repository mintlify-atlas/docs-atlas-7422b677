The type conversion system in Skript Parser provides automatic conversion between different types. It supports direct conversions, chained conversions, and automatic creation of missing converter chains.

## Converters Class

The main class for managing type conversions.

```java
public abstract class Converters
```

### Constants

```java
public static final int ALL_CHAINING = 0;
public static final int NO_LEFT_CHAINING = 1;
public static final int NO_RIGHT_CHAINING = 2;
public static final int NO_CHAINING = NO_LEFT_CHAINING | NO_RIGHT_CHAINING;
```

<ResponseField name="ALL_CHAINING" type="int">
  Flag declaring a converter may freely be part of a ChainedConverter
</ResponseField>

<ResponseField name="NO_LEFT_CHAINING" type="int">
  Flag declaring a converter may only be the first part of a ChainedConverter
</ResponseField>

<ResponseField name="NO_RIGHT_CHAINING" type="int">
  Flag declaring a converter may only be the second part of a ChainedConverter
</ResponseField>

<ResponseField name="NO_CHAINING" type="int">
  Flag declaring a converter may not be part of a ChainedConverter at all
</ResponseField>

## ChainedConverter

Automatically created converter that chains two converters together.

```java
public final class ChainedConverter<F, M, T> implements Function<F, Optional<? extends T>> {
    public ChainedConverter(
        Function<? super F, Optional<? extends M>> first,
        Function<? super M, Optional<? extends T>> second
    )
}
```

<ParamField path="F" type="type parameter">
  The source type (from)
</ParamField>

<ParamField path="M" type="type parameter">
  The middle type (intermediate conversion result)
</ParamField>

<ParamField path="T" type="type parameter">
  The target type (to)
</ParamField>

<ParamField path="first" type="Function<? super F, Optional<? extends M>>" required>
  The first converter (F → M)
</ParamField>

<ParamField path="second" type="Function<? super M, Optional<? extends T>>" required>
  The second converter (M → T)
</ParamField>

### How It Works

If you have:
- Converter A: `String → Integer`
- Converter B: `Integer → Double`

A ChainedConverter automatically creates: `String → Double`

## Example Usage

### Registering a Simple Converter

```java
// Convert String to Integer
Converters.registerConverter(
    String.class,
    Integer.class,
    str -> {
        try {
            return Optional.of(Integer.parseInt(str));
        } catch (NumberFormatException e) {
            return Optional.empty();
        }
    }
);
```

### Registering with Chaining Flags

```java
// This converter cannot be chained
Converters.registerConverter(
    CustomType.class,
    String.class,
    obj -> Optional.of(obj.toString()),
    Converters.NO_CHAINING
);

// This converter can only be first in a chain
Converters.registerConverter(
    String.class,
    CustomType.class,
    str -> Optional.of(new CustomType(str)),
    Converters.NO_RIGHT_CHAINING
);
```

### Using Converters

```java
// Convert a single value
String str = "42";
Optional<Integer> num = Converters.convert(str, Integer.class);
if (num.isPresent()) {
    System.out.println("Converted: " + num.get());
}

// Convert to multiple possible types
Object value = "test";
Optional<?> result = Converters.convert(
    value,
    new Class[]{Integer.class, Boolean.class, String.class}
);

// Convert an array
String[] strings = {"1", "2", "3"};
Optional<Integer[]> numbers = Converters.convertArray(strings, Integer.class);
```

### Using a Specific Converter

```java
// Get the converter once for better performance
Optional<? extends Function<? super String, Optional<? extends Integer>>> converter =
    Converters.getConverter(String.class, Integer.class);

if (converter.isPresent()) {
    String[] inputs = {"1", "2", "3", "invalid", "5"};
    
    for (String input : inputs) {
        Optional<? extends Integer> result = converter.get().apply(input);
        result.ifPresent(num -> System.out.println("Converted: " + num));
    }
}
```

### Batch Registration

```java
SkriptRegistration registration = new SkriptRegistration();

// Register multiple converters
registration.addConverter(new ConverterInfo<>(
    String.class,
    Integer.class,
    str -> {
        try {
            return Optional.of(Integer.parseInt(str));
        } catch (NumberFormatException e) {
            return Optional.empty();
        }
    }
));

registration.addConverter(new ConverterInfo<>(
    Integer.class,
    String.class,
    Object::toString
));

// Register all at once
Converters.registerConverters(registration);

// Create chained converters
Converters.createMissingConverters();
```

### Creating Automatic Converter Chains

```java
// Register individual converters
Converters.registerConverter(String.class, Integer.class, str -> {
    try {
        return Optional.of(Integer.parseInt(str));
    } catch (NumberFormatException e) {
        return Optional.empty();
    }
});

Converters.registerConverter(Integer.class, Double.class,
    i -> Optional.of(i.doubleValue())
);

Converters.registerConverter(Double.class, String.class,
    d -> Optional.of(String.format("%.2f", d))
);

// Automatically create chains:
// String → Integer → Double
// Integer → Double → String
// String → Integer → Double → String (full chain)
Converters.createMissingConverters();

// Now you can convert String directly to Double
Optional<Double> result = Converters.convert("42", Double.class);
// This uses the chained converter: String → Integer → Double
```

## Best Practices

### Return Optional.empty() for Failed Conversions

Always return `Optional.empty()` instead of throwing exceptions:

```java
// Good
Converters.registerConverter(String.class, Integer.class, str -> {
    try {
        return Optional.of(Integer.parseInt(str));
    } catch (NumberFormatException e) {
        return Optional.empty();
    }
});

// Avoid
Converters.registerConverter(String.class, Integer.class, str -> {
    return Optional.of(Integer.parseInt(str)); // Throws exception!
});
```

### Use Chaining Flags Appropriately

Set chaining flags based on the converter's behavior:

```java
// Lossy conversion - prevent further chaining
Converters.registerConverter(
    Double.class,
    Integer.class,
    d -> Optional.of(d.intValue()),
    Converters.NO_RIGHT_CHAINING
);

// Expensive conversion - prevent chaining
Converters.registerConverter(
    String.class,
    ComplexObject.class,
    str -> parseComplex(str),
    Converters.NO_CHAINING
);
```

### Check if Converter Exists

Before attempting conversion, check if a converter exists:

```java
if (Converters.converterExists(String.class, CustomType.class)) {
    Optional<CustomType> result = Converters.convert("value", CustomType.class);
}
```

### Reuse Converters for Performance

When converting many values, get the converter once:

```java
// Efficient
var converter = Converters.getConverter(String.class, Integer.class);
for (String str : manyStrings) {
    converter.flatMap(c -> c.apply(str)).ifPresent(this::process);
}

// Less efficient
for (String str : manyStrings) {
    Converters.convert(str, Integer.class).ifPresent(this::process);
}
```

### Call createMissingConverters() After Registration

Always call this after registering all converters:

```java
// Register all converters
Converters.registerConverter(A.class, B.class, converterAB);
Converters.registerConverter(B.class, C.class, converterBC);
Converters.registerConverter(C.class, D.class, converterCD);

// Create chains (A→B→C, B→C→D, A→B→C→D, etc.)
Converters.createMissingConverters();
```
