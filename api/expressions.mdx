## Overview

The `Expression<T>` interface represents a syntax element that returns a value of a specific type. All expressions in Skript extend this interface.

**Package:** `io.github.syst3ms.skriptparser.lang`

**Type Parameter:**
- `T` - the type of value this expression returns

**Extends:** `SyntaxElement`

## Interface Methods

### Core Value Retrieval

#### getValues()

```java
T[] getValues(TriggerContext ctx)
```

Retrieves all values of this Expression, accounting for possible modifiers. If this is an or-list, it will choose a random value to return.

<ParamField path="ctx" type="TriggerContext">
  The trigger context
</ParamField>

<ResponseField name="return" type="T[]">
  An array of the values, or null if no values exist
</ResponseField>

---

#### getArray()

```java
default T[] getArray(TriggerContext ctx)
```

Retrieves all values of this Expression without accounting for modifiers. If this is an or-list, it will still return all possible values.

<ParamField path="ctx" type="TriggerContext">
  The trigger context
</ParamField>

<ResponseField name="return" type="T[]">
  An array of the raw values (never null, empty array if no values)
</ResponseField>

---

#### getSingle()

```java
default Optional<? extends T> getSingle(TriggerContext ctx)
```

Gets a single value out of this Expression.

<ParamField path="ctx" type="TriggerContext">
  The trigger context
</ParamField>

<ResponseField name="return" type="Optional<? extends T>">
  The single value, or empty if it has no value
</ResponseField>

<Warning>
  Throws `SkriptRuntimeException` if the expression returns more than one value
</Warning>

### Change Support

#### acceptsChange(ChangeMode)

```java
default Optional<Class<?>[]> acceptsChange(ChangeMode mode)
```

Determines whether this expression can be changed according to a specific `ChangeMode`, and what type of values it can be changed with.

<ParamField path="mode" type="ChangeMode">
  The mode this Expression would be changed with
</ParamField>

<ResponseField name="return" type="Optional<Class<?>[]>">
  An array of classes describing what types this Expression can be changed with, empty if it shouldn't be changed with the given change mode. For DELETE or RESET modes, an empty array should be returned.
</ResponseField>

## Implementation Example

```java
public class ExprLoopValue extends SectionValue<SecLoop, Object> {
    private SecLoop loop;
    private boolean isVariableLoop;
    private boolean isIndex;
    private Class<?> loopedClass;
    
    @Override
    public Object[] getSectionValues(SecLoop loop, TriggerContext ctx) {
        Object[] one = (Object[]) Array.newInstance(getReturnType(), 1);
        if (isVariableLoop) {
            if (loop.getArguments() == null || loop.getArguments()[0] == null) {
                return new Object[0];
            }
            var current = (Pair<String, Object>) loop.getArguments()[0];
            if (isIndex) {
                return new String[]{current.getFirst()};
            }
            one[0] = current.getSecond();
            return one;
        }
        Object[] arguments = loop.getArguments();
        if (arguments == null || arguments.length == 0) {
            return new Object[0];
        }
        one[0] = loop.getArguments()[0];
        return one;
    }
    
    @Override
    public Class<?> getReturnType() {
        if (isIndex)
            return String.class;
        return loop.getLoopedExpression().getReturnType();
    }
    
    @Override
    public String toString(TriggerContext ctx, boolean debug) {
        if (loop == null)
            return "loop-" + loopedString;
        if (isVariableLoop) {
            var variable = (Variable<?>) loop.getLoopedExpression();
            return isIndex ? 
                "\"" + variable.getIndex(ctx) + "\"" : 
                variable.toString(ctx, debug);
        }
        return loop.getLoopedExpression().toString(ctx, debug);
    }
}
```
