The type system in Skript Parser provides a robust framework for defining, managing, and working with types. It includes support for plural forms, literal parsing, type conversion, and change operations.

## Type Class

The `Type<T>` class is the basic definition of a type in Skript Parser.

### Constructor

```java
public Type(
    Class<T> typeClass,
    String baseName,
    String pattern,
    @Nullable Function<String, ? extends T> literalParser,
    Function<? super T, String> toStringFunction,
    @Nullable Function<? super T, String> toVariableNameFunction,
    @Nullable Changer<? super T> defaultChanger,
    @Nullable Arithmetic<T, ?> arithmetic,
    Documentation documentation,
    @Nullable TypeSerializer<T> serializer,
    @Nullable Supplier<Iterator<T>> supplier
)
```

<ParamField path="typeClass" type="Class<T>" required>
  The Java class this type represents
</ParamField>

<ParamField path="baseName" type="String" required>
  The basic name to represent this type with. Should be a lowercase version of the Java class name.
</ParamField>

<ParamField path="pattern" type="String" required>
  The pattern for plural forms in Skript aliases plural format:
  - `fish` → "fish" (invariant)
  - `dog|s` → "dog" and "dogs"
  - `part|y|ies` → "party" and "parties" (irregular plural)
</ParamField>

<ParamField path="literalParser" type="Function<String, ? extends T>">
  Function that parses string literals into instances of this type. If the parser throws an exception, it will be caught and the type will be ignored.
</ParamField>

<ParamField path="toStringFunction" type="Function<? super T, String>">
  Function that converts an object of type T to a String. Defaults to `Objects::toString`.
</ParamField>

<ParamField path="toVariableNameFunction" type="Function<? super T, String>">
  Function that converts an object to a variable name representation.
</ParamField>

<ParamField path="defaultChanger" type="Changer<? super T>">
  The default changer for this type, defining how values can be modified.
</ParamField>

<ParamField path="arithmetic" type="Arithmetic<T, ?>">
  Arithmetic operations support for this type.
</ParamField>

<ParamField path="documentation" type="Documentation" required>
  Documentation information for this type.
</ParamField>

<ParamField path="serializer" type="TypeSerializer<T>">
  Serializer for persisting values of this type.
</ParamField>

<ParamField path="supplier" type="Supplier<Iterator<T>>">
  Supplier that provides an iterator over all possible values of this type.
</ParamField>

## PatternType

A type used in a pattern that groups a Type and a number (single or plural) together.

```java
public final class PatternType<T> {
    public PatternType(Type<T> type, boolean single)
    public Type<T> getType()
    public boolean isSingle()
}
```

<ParamField path="type" type="Type<T>" required>
  The underlying type
</ParamField>

<ParamField path="single" type="boolean" required>
  Whether this represents the singular form (true) or plural form (false)
</ParamField>

## ChangeMode

An enum representing how an expression can be changed.

```java
public enum ChangeMode {
    SET,           // Expression is being set to one or more values
    ADD,           // Values are being added to expression
    REMOVE,        // Values are being removed from expression
    DELETE,        // Expression is being deleted
    RESET,         // Expression is being reset to default value
    REMOVE_ALL     // All matching values are being removed
}
```

### Values

<ResponseField name="SET" type="ChangeMode">
  Indicates that an expression is being set to one or more values
</ResponseField>

<ResponseField name="ADD" type="ChangeMode">
  Indicates that one or more values are being added to an expression
</ResponseField>

<ResponseField name="REMOVE" type="ChangeMode">
  Indicates that one or more values are being removed from an expression
</ResponseField>

<ResponseField name="DELETE" type="ChangeMode">
  Indicates that an expression is being deleted
</ResponseField>

<ResponseField name="RESET" type="ChangeMode">
  Indicates that an expression is being reset to a default value. This is NOT equivalent to DELETE.
</ResponseField>

<ResponseField name="REMOVE_ALL" type="ChangeMode">
  Indicates that all values matching the given expression are being removed. This is NOT equivalent to DELETE. For example, remove all values of a specific type from a list.
</ResponseField>

## Example Usage

### Registering a Custom Type

```java
// Define a simple type with plural forms
Type<CustomObject> customType = new Type<>(
    CustomObject.class,
    "customobject",
    "custom object|s",
    str -> Optional.of(new CustomObject(str)),
    Documentation.fromString("A custom object type"),
    null
);

// Register it
SkriptRegistration registration = new SkriptRegistration();
registration.addType(customType);
TypeManager.register(registration);
```

### Using TypeManager

```java
// Look up a type by name
Optional<? extends Type<?>> numberType = TypeManager.getByName("number");

// Look up by class
Optional<? extends Type<String>> stringType = TypeManager.getByClassExact(String.class);

// Get a pattern type (with singular/plural info)
Optional<PatternType<?>> patternType = TypeManager.getPatternType("numbers");
if (patternType.isPresent()) {
    boolean isPlural = !patternType.get().isSingle(); // true for "numbers"
}

// Convert objects to string
Object[] values = {1, 2, 3};
String result = TypeManager.toString(values); // "1, 2 and 3"
```

### Implementing a Changer

```java
public class CustomChanger implements Changer<CustomObject> {
    @Override
    public Class<?>[] acceptsChange(ChangeMode mode) {
        return switch (mode) {
            case SET -> new Class[]{String.class};
            case ADD -> new Class[]{Integer.class};
            case DELETE, RESET -> new Class[0];
            default -> null;
        };
    }

    @Override
    public void change(CustomObject[] toChange, Object[] changeWith, ChangeMode mode) {
        switch (mode) {
            case SET -> {
                for (CustomObject obj : toChange) {
                    obj.setValue((String) changeWith[0]);
                }
            }
            case ADD -> {
                for (CustomObject obj : toChange) {
                    obj.increment((Integer) changeWith[0]);
                }
            }
            case DELETE -> {
                for (CustomObject obj : toChange) {
                    obj.clear();
                }
            }
        }
    }
}
```
