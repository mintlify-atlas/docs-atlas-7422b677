## Overview

The `SkriptRegistration` class is a mutable object that tracks all syntax and types registered by a `SkriptAddon`. It provides a fluent API for registering expressions, effects, sections, events, types, converters, and more.

<Warning>
  Don't forget to call `register()` after adding all your syntax elements!
</Warning>

## Complete Example

```java
public class MyAddon extends SkriptAddon {
    private final SkriptRegistration registration;
    
    public MyAddon() {
        super("MyAddon");
        registration = new SkriptRegistration(this);
        
        // Register a type
        registration.newType(CustomObject.class, "customobject", "custom object[s]")
            .toStringFunction(obj -> obj.getName())
            .literalParser(str -> CustomObject.parse(str))
            .register();
        
        // Register an expression
        registration.newExpression(
                CustomExpression.class,
                String.class,
                true,
                "custom value of %customobject%"
            )
            .setPriority(5)
            .register();
        
        // Register an effect
        registration.addEffect(
            CustomEffect.class,
            "do something with %customobject%"
        );
        
        // Register a converter
        registration.addConverter(
            CustomObject.class,
            String.class,
            obj -> Optional.of(obj.toString())
        );
        
        // Register an event
        registration.newEvent(CustomEvent.class, "custom event")
            .setHandledContexts(CustomContext.class)
            .register();
        
        // Finalize registration
        List<LogEntry> logs = registration.register();
        
        // Handle any errors
        for (LogEntry log : logs) {
            if (log.getType() == ErrorType.ERROR) {
                System.err.println("Registration error: " + log.getMessage());
            }
        }
    }
    
    @Override
    public SkriptRegistration getSkriptRegistration() {
        return registration;
    }
}
```

## Builder Pattern Example

```java
// Using the builder pattern for complex registration
registration.newExpression(
        PlayerHealthExpression.class,
        Number.class,
        true,
        "health of %players%",
        "%players%'[s] health"
    )
    .setPriority(3)
    .addData("canModify", true)
    .register();

// Property expression with builder
registration.newPropertyExpression(
        ItemNameProperty.class,
        String.class,
        "name",
        "%itemstacks%"
    )
    .setPriority(4)
    .register();
```

## Best Practices

1. **Always call register()**: Don't forget to call `register()` after adding all syntax elements
2. **Use builders for complex syntax**: The builder pattern provides better control and readability
3. **Handle registration logs**: Check for errors in the returned `LogEntry` list
4. **Set appropriate priorities**: Lower priority syntax should have higher numbers (default is 5)
5. **Register types first**: Always register types before expressions that use them
