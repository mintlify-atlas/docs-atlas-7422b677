# Quick Start Guide

This guide will walk you through creating a minimal working example that initializes Skript Parser and executes a simple script.

## Overview

Using Skript Parser involves three main steps:

1. **Initialize** the parser with `Parser.init()`
2. **Create** a Skript file (`.sk` extension)
3. **Run** the script with `Parser.run()`

## Step-by-Step Tutorial

<Steps>
  <Step title="Create your main class">
    Create a new Java class that will initialize and run the parser:
    
    ```java Main.java
    import io.github.syst3ms.skriptparser.Parser;
    
    public class Main {
        public static void main(String[] args) {
            // Initialize the parser
            Parser.init(
                new String[0],      // Main packages to load
                new String[0],      // Sub-packages to load
                new String[0],      // Program arguments
                true                // Standalone mode (enables addon loading)
            );
            
            // Run a script file
            Parser.run(
                "script.sk",        // Script file name
                false,              // Debug mode
                true                // Enable tips
            );
        }
    }
    ```
    
    <Note>
      The `Parser.init()` method loads all default syntax, types, and registers built-in features like integers, strings, booleans, and more.
    </Note>
  </Step>
  
  <Step title="Create a Skript file">
    Create a file named `script.sk` in the same directory as your JAR file (or provide the full path):
    
    ```vba script.sk
    on test:
        log "Hello from Skript Parser!"
        set {_number} to 42
        log "The number is: " & {_number}
    ```
    
    This simple script demonstrates:
    - Event handling (`on test`)
    - Logging output
    - Variable assignment
    - String concatenation
  </Step>
  
  <Step title="Run your application">
    Compile and run your Java application:
    
    ```bash
    javac Main.java
    java Main
    ```
    
    You should see output indicating the script was loaded successfully, along with any log messages from your script.
  </Step>
</Steps>

## Understanding the Parser API

### Parser.init()

The `Parser.init()` method initializes the parser and loads all syntax elements:

```java
public static void init(
    String[] mainPackages,   // Packages to search for syntax classes
    String[] subPackages,    // Subpackages within main packages
    String[] programArgs,    // Arguments to pass to the Skript instance
    boolean standalone       // Whether to load addons from /addons folder
)
```

**Parameters:**

- `mainPackages`: Additional packages to scan for syntax classes (beyond the built-in `io.github.syst3ms.skriptparser`)
- `subPackages`: Additional subpackages like "expressions", "effects", "event", etc.
- `programArgs`: Command-line arguments available to your scripts
- `standalone`: If `true`, loads addon JARs from an `addons/` folder next to your application

<Note>
  Skript Parser automatically loads its own core syntax from `io.github.syst3ms.skriptparser` with subpackages: `expressions`, `effects`, `event`, `lang`, `sections`, `structures`, and `tags`.
</Note>

### Parser.run()

The `Parser.run()` method parses and executes a script file:

```java
public static void run(
    String scriptName,      // Path to the script file
    boolean debug,          // Enable debug logging
    boolean tipsEnabled     // Show helpful tips in error messages
)
```

**Parameters:**

- `scriptName`: Path to the `.sk` file (relative or absolute)
- `debug`: Enables verbose debug output for troubleshooting
- `tipsEnabled`: When `true`, shows helpful tips alongside error messages

## Working with the Registration API

Skript Parser uses a registration system to define custom syntax. Here's how to register a custom type:

```java
import io.github.syst3ms.skriptparser.Parser;
import io.github.syst3ms.skriptparser.registration.SkriptRegistration;

public class CustomSetup {
    public static void main(String[] args) {
        // Get the main registration instance
        SkriptRegistration registration = Parser.getMainRegistration();
        
        // Register a custom type
        registration.newType(Integer.class, "integer", "integer@s")
            .literalParser(s -> {
                try {
                    return Integer.parseInt(s);
                } catch (NumberFormatException e) {
                    return null;
                }
            })
            .name("Integer")
            .description("A whole number.")
            .since("1.0.0")
            .register();
        
        // Initialize and run
        Parser.init(new String[0], new String[0], args, true);
        Parser.run("script.sk", false, true);
    }
}
```

<Warning>
  Custom registrations must happen **before** calling `Parser.init()`, or you need to get the registration object and call `.register()` manually.
</Warning>

## Built-in Types

Skript Parser comes with many built-in types registered by default:

- **Numbers**: `integer`, `float`, `double`, `number`
- **Text**: `string`, `boolean`
- **Time**: `duration`, `date`, `time`
- **Other**: `object`, `type`, `color`

All these types are automatically available in your scripts through the `DefaultRegistration.register()` call that happens during initialization.

## Command-Line Usage

Skript Parser also includes a built-in CLI for running scripts directly:

```bash
java -jar skript-parser.jar script.sk
```

With options:

```bash
java -jar skript-parser.jar --debug --no-tips script.sk arg1 arg2
```

**Available flags:**
- `--debug`: Enable debug mode
- `--no-tips` or `--nt`: Disable helpful tips in error messages
- Any arguments after the script name are passed as program arguments

## Error Handling

Skript Parser provides detailed error messages with line numbers and helpful tips:

```
[12:34:56] ERROR: Expected 'on', 'every', or 'at' but found 'when'
[12:34:56] TIP: Events must start with 'on', 'every', or 'at'. Did you mean 'on test'?
```

When `tipsEnabled` is `true`, you'll receive contextual suggestions to fix common mistakes.

## Complete Example

Here's a complete, runnable example:

```java CompleteExample.java
import io.github.syst3ms.skriptparser.Parser;

public class CompleteExample {
    public static void main(String[] args) {
        // Initialize the parser with default settings
        Parser.init(
            new String[0],
            new String[0],
            args,
            true
        );
        
        // Run the script
        Parser.run("example.sk", false, true);
    }
}
```

```vba example.sk
on test:
    log "Skript Parser is working!"
    
    set {_items} to 1, 2, 3, 4, 5
    loop {_items}:
        log "Item: " & loop-value
    
    if 10 > 5:
        log "Math works!"
```

Compile and run:

```bash
javac -cp ".:skript-parser-1.0.11.jar" CompleteExample.java
java -cp ".:skript-parser-1.0.11.jar" CompleteExample
```

<Note>
  On Windows, use semicolons instead of colons in the classpath: `-cp ".;skript-parser-1.0.11.jar"`
</Note>
